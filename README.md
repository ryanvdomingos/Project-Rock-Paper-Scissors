# ✊✋✌️ Pedra, Papel e Tesoura (Rock, Paper, Scissors)

🔗 **[Clique aqui para jogar a versão ao vivo!](https://ryanvdomingos.github.io/Project-Rock-Paper-Scissors/)**

## 📖 Sobre o Projeto
Um clássico jogo de Pedra, Papel e Tesoura construído com JavaScript, HTML e CSS. 

Este projeto faz parte da trilha de Fundamentos do **[The Odin Project](https://www.theodinproject.com/)**. O objetivo principal foi colocar em prática os fundamentos do JavaScript, começando com um jogo simples rodando no Console do navegador e, posteriormente, evoluindo o projeto para incluir uma Interface Gráfica de Usuário (GUI) completa manipulando o DOM.

## ✨ Funcionalidades
* O usuário joga contra o computador, que faz escolhas baseadas em cálculos aleatórios (`Math.random`).
* Interface interativa com botões para escolher Pedra, Papel ou Tesoura.
* Um placar em tempo real acompanha as vitórias do jogador e do computador.
* O jogo anuncia o vencedor da rodada e declara o Grande Campeão assim que alguém atingir **5 pontos**.

## 🚀 Tecnologias Utilizadas
* **HTML5:** Estruturação dos botões e da área de exibição de resultados.
* **CSS3:** Estilização básica da interface.
* **JavaScript (Vanilla):** Lógica do jogo, sorteio do computador e manipulação do DOM.

## 🧠 O que eu aprendi
Este projeto foi um grande marco na compreensão de como o JavaScript interage com a página web. Os principais aprendizados incluem:
* **Manipulação do DOM:** Capturar elementos do HTML usando `document.querySelector` e alterar textos dinamicamente.
* **Event Listeners:** Utilização do `addEventListener` para escutar cliques nos botões e disparar funções de forma interativa.
* **Mudança de Paradigma:** Transição de um código procedural e engessado (usando `prompt` e laços `for`) para uma arquitetura orientada a eventos (Event-Driven), onde o fluxo do jogo é ditado pelas ações do usuário.
* Escopo de variáveis, parâmetros de funções e a importância de manter o código limpo e modular.
